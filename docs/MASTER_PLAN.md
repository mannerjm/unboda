# PART 1. 프로젝트 비전

운보다는 단순히 사주 결과를 보여주는 서비스가 아니다.

사주를 중심으로 시작하여
궁합, 택일, 작명·개명, 이사, 개업·창업,
타로, 풍수지리 등 개인의 삶과 선택에 관련된 다양한 운세 콘텐츠를
하나의 AI 경험으로 연결하는 통합 운명 분석 플랫폼을 목표로 한다.

운보다의 핵심 목표:

- 신뢰할 수 있는 명리 계산
- 사용자가 이해하기 쉬운 해석
- 계산과 AI 해석의 명확한 분리
- 개인의 현재 상황과 연결되는 분석
- 불안을 과도하게 조장하지 않는 상담
- 실제 생활과 의사결정에 도움이 되는 정보
- 모바일 중심의 편리한 사용자 경험
- 한 번 보고 끝나지 않는 지속적인 운의 관리
- 사용자의 필요에 맞는 분석을 먼저 발견하는 개인화

운보다가 궁극적으로 지향하는 것은

"사주를 보여주는 서비스"

가 아니라

"현재 나에게 어떤 변화가 일어나고 있으며,
무엇을 살펴볼 필요가 있는지를 이해하도록 돕는
개인 운명 분석 플랫폼"

이다.

# PART 2. 절대 제품 원칙

## 2-1. Engine First

모든 명리 계산은 엔진에서 처리한다.

UI에서는 명리 계산을 하지 않는다.

AI는 사주를 임의로 계산하지 않는다.

기본 구조:

사용자 입력
→ 계산 엔진
→ 구조화된 분석 데이터
→ AI 해석
→ 사용자 결과

AI는 가능한 한
엔진에서 계산된 사실 데이터를 근거로 설명한다.

---

## 2-2. 계산과 해석의 분리

계산 엔진의 역할:

- 원국 계산
- 오행 분석
- 십신 분석
- 신강 / 신약
- 용신
- 격국
- 대운
- 세운
- 합·충·형·파·해
- 시기별 변화 데이터

AI의 역할:

- 계산 결과 설명
- 여러 계산 결과의 연결
- 사용자의 질문과 분석 결과 연결
- 이해하기 쉬운 자연어 해석
- 현실적인 행동 가이드 제공

AI가 계산 결과와 충돌하는 내용을
임의로 생성하지 않도록 설계한다.

---

## 2-3. 기능별 엔진 분리

기능은 가능한 한 독립적인 엔진으로 분리한다.

예시:

app/lib
├─ manse.ts
├─ nobles.ts
├─ elements.ts
├─ weights.ts
├─ strength.ts
├─ yongsin.ts
├─ daeun.ts
├─ sewoon.ts
├─ compatibility.ts
├─ recommendation.ts
├─ tarot.ts
└─ dream.ts

실제 파일명과 구조는 구현 과정에서 변경할 수 있지만,
계산 로직을 UI와 분리한다는 원칙은 유지한다.

---

## 2-4. 무료와 유료의 품질 원칙

무료 분석의 품질을 의도적으로 낮추지 않는다.

무료 분석에서도 사용자가

"이 서비스는 나를 제대로 분석하고 있다."

라고 느낄 수 있어야 한다.

무료와 유료의 차이는
정확도의 차이가 아니라
분석의 깊이와 문제 해결 범위의 차이로 설계한다.

무료:

"현재 어떤 흐름인가?"

유료:

"왜 이런 흐름이 발생하는가?"
"언제 강해지는가?"
"어떤 기회와 위험이 있는가?"
"어떤 선택을 고려할 수 있는가?"
"현재 상황에서 어떻게 대응할 것인가?"

---

## 2-5. 불안 기반 판매 금지

사용자의 불안을 과도하게 자극해
결제를 유도하지 않는다.

예:

"큰 사고가 생길 수 있습니다. 결제하세요."

같은 구조를 사용하지 않는다.

대신 실제 분석 결과를 기반으로

"현재 어떤 변화가 나타나는가"
"왜 추가 분석이 도움이 될 수 있는가"

를 설명한다.

# PART 4. 사용자·회원·프로필 운영 정책

## 4-1. 회원가입과 구독은 분리한다

회원가입과 유료 구독은 같은 개념이 아니다.

회원가입:

사용자의 데이터와 구매 자산을 보관하기 위한 계정

구독:

지속적인 프리미엄 기능을 이용하기 위한 선택형 유료 서비스

사용자는 다음 상태로 존재할 수 있다.

1. 비회원
2. 무료회원
3. 단품 구매회원
4. 구독회원

단품 유료 분석은
구독 없이 구매할 수 있다.

---

## 4-2. 최초 무료 분석

최초 접속 시 회원가입을 강제하지 않는다.

기본 흐름:

운보다 접속
→ 회원가입 없이 사주정보 입력
→ 무료 분석
→ 결과 확인
→ 개인 맞춤 분석 추천

무료 결과를 보기 위해
회원가입을 필수로 요구하지 않는다.

---

## 4-3. 무료 결과 저장

무료 분석 후에는
자발적인 회원가입 전환 포인트를 제공한다.

예:

"내 사주 저장하기"

로그인하면 입력한 사주와 분석 결과를 저장하고
다음 방문부터 다시 입력하지 않아도 되도록 한다.

---

## 4-4. 단품 결제 시 회원 연결

사용자가 유료 분석을 구매하려는 경우
결제 과정에서 계정을 연결한다.

기본 흐름:

무료 분석
→ 유료 분석 선택
→ 간편 로그인 / 회원가입
→ 결제
→ 분석 생성
→ 구매 결과 영구 저장

회원가입은 단순한 결제 장벽이 아니라
구매한 분석 결과를 안전하게 보관하기 위한 구조로 설계한다.

---

## 4-5. User와 Profile 분리

시스템 설계에서

User = 서비스 계정

Profile = 사주 분석 대상

으로 분리한다.

한 명의 User는
여러 개의 Profile을 가질 수 있다.

예:

User
├─ 본인 Profile
├─ 배우자 Profile
├─ 자녀 Profile
└─ 가족 / 지인 Profile

중요:

프로필 저장 권한과
유료 분석 이용 권한은 분리한다.

프로필을 저장했다고 해서
해당 인물의 모든 유료 분석이
자동으로 무료 제공되는 것은 아니다.

---

## 4-6. 권한 모델

향후 시스템은 다음 개념을 분리해 관리한다.

User
→ 사용자 계정

Profile
→ 독립적인 분석 대상

Entitlement
→ 특정 분석 콘텐츠 이용 권한

Purchase
→ 단품 구매 기록

Subscription
→ 지속적인 구독 혜택

Credit
→ AI 질문 또는 추가 분석 등에 사용할 수 있는 이용권

이를 통해

무료회원
단품 구매회원
구독회원
가족 플랜

등을 유연하게 운영할 수 있도록 한다.

운보다의 수익 모델은
하나의 구독 상품에 의존하지 않는다.

기본 구조:

단품 분석
+
AI 명리 질문
+
선택형 구독
+
관계 기반 상품
+
전문 명리 서비스

의 하이브리드 모델로 운영한다.

---

## 5-1. 무료 분석

무료 분석의 역할은 신뢰 형성이다.

예:

- 기본 사주 원국
- 오행 분포
- 신강 / 신약
- 용신 요약
- 현재 대운 요약
- 현재 세운 요약
- 현재 가장 중요한 운의 흐름

구체적인 무료 범위는
실제 사용자 반응과 전환 데이터를 기반으로 조정한다.

---

## 5-2. 고민별 단품 심층 분석

운보다의 초기 핵심 매출 모델이다.

예:

- 재물운 심층 분석
- 직업 / 사업운
- 연애 / 관계 분석
- 올해 운세
- 10년 대운 분석
- 종합 인생 리포트

사용자는 구독하지 않아도
필요한 분석만 구매할 수 있다.

---

## 5-3. 유료 리포트 기본 구조

1. 현재 상황
2. 명리학적 근거
3. 대운 흐름
4. 세운 흐름
5. 중요한 시기
6. 기회 요인
7. 주의 요인
8. 현실적인 행동 가이드

---

## 5-4. AI 명리 질문

예:

- 올해 이직해도 될까요?
- 사업을 시작해도 될까요?
- 지금 투자해도 될까요?
- 이 사람과 결혼해도 될까요?
- 언제 움직이는 것이 좋을까요?

AI 상담은 가능한 한 다음 데이터를 기반으로 한다.

- 원국
- 신강 / 신약
- 용신
- 대운
- 세운
- 합·충·형·파·해
- 오행 변화
- 기존 분석 결과

수익 모델:

- 질문 단품
- 질문 묶음
- 구독 혜택

---

## 5-5. 선택형 구독

구독은 필수가 아니다.

구독은 반복적으로 운보다를 사용하는 사용자에게
지속적인 가치를 제공하는 선택형 서비스다.

예시:

운보다 Plus

- 일간 / 월간 운세
- 운의 변화 추적
- AI 질문 혜택
- 변화 알림
- 일부 지속형 분석

운보다 Premium

- 확대된 AI 상담
- 추가 프로필 관련 혜택
- 관계 분석 혜택
- 중요 시기 분석
- 고급 의사결정 분석

구독 하나로
모든 프로필과 모든 전문 서비스를
무제한 제공하지 않는다.

구체적인 가격과 제공량은
실제 운영 데이터를 기반으로 결정한다.

---

## 5-6. 추가 인물과 관계 상품

새로운 Profile은
새로운 분석 가능성을 만든다.

예:

- 배우자 심층 분석
- 자녀 성향 / 진로 분석
- 연인 궁합
- 부부 궁합
- 부모·자녀 관계
- 사업 파트너 궁합
- 가족 전체 흐름

가족 프로필 저장과
가족의 유료 심층 분석 권한은 분리한다.

# PART 7. 개인 맞춤 추천 시스템

향후 recommendationEngine을 구축한다.

목표:

모든 사용자에게 같은 상품을 보여주는 것이 아니라
현재 사주와 운의 흐름을 분석해
가장 관련성이 높은 다음 분석을 추천한다.

입력 예시:

- 원국
- 오행
- 신강 / 신약
- 용신
- 대운
- 세운
- 합·충·형·파·해
- fortuneFlowAnalysis
- 사용자 관심사
- 기존 구매 이력

출력 예시:

- 현재 가장 중요한 운
- 주요 변화 신호
- 추천 분석 상품
- 추천 이유
- 추천 우선순위
- 재방문 권장 시점

예:

직업 변화 신호 강함
→ 직업 / 사업운 분석

재물 변화 신호 강함
→ 재물운 심층 분석

관계 변화 신호 강함
→ 연애 / 관계 분석

대운 전환기 접근
→ 10년 대운 분석

중요:

추천 엔진은
가장 비싼 상품을 추천하는 판매 엔진이 아니다.

실제 계산 결과와 사용자의 현재 필요를 기준으로
관련성이 높은 분석을 추천한다.

# PART 8. 재방문 엔진

운보다의 장기 경쟁력은
시간에 따라 변화하는 운을 지속적으로 연결하는 것이다.

감지 대상:

- 월운 변화
- 세운 변화
- 대운 전환
- 용신 활성 시기
- 중요한 합·충·형·파·해
- 직업 변화 신호
- 재물 변화 신호
- 관계 변화 신호

기본 흐름:

의미 있는 운 변화 발생
→ 사용자 알림
→ 운보다 재방문
→ 변화 핵심 무료 요약
→ 필요한 심층 분석 추천

알림은 실제 계산 결과를 기반으로 하며
불안을 과도하게 자극하지 않는다.

# PART 9. 마케팅·사용자 획득·바이럴 성장 전략

## 9-1. 핵심 성장 원칙

운보다는

"서비스를 만든 뒤 광고하는 플랫폼"

으로 개발하지 않는다.

제품 자체에

- 사용하고 싶은 이유
- 공유하고 싶은 이유
- 다시 돌아올 이유
- 결제할 이유

를 함께 설계한다.

---

## 9-2. 출시 전 베타

정식 출시 전
소규모 실제 사용자를 대상으로 검증한다.

핵심 지표:

- 사주 입력 시작률
- 입력 완료율
- 무료 분석 완료율
- 결과 열람률
- 회원가입률
- 저장 전환율
- 추천 상품 클릭률
- 결제 시작률
- 결제 완료율
- 상품별 구매율
- 공유율
- 재방문율

초기 목적은 많은 사용자가 아니라
실제 사용자의 행동을 이해하는 것이다.

## 9-3. 공유 콘텐츠

예:

- 핵심 오행 카드
- 사주 성향 카드
- 올해의 핵심 운
- 현재 가장 중요한 운
- 운의 변화 카드
- 궁합 카드

공유 콘텐츠에는
생년월일과 출생시간 등
민감한 개인정보를 노출하지 않는다.

---

## 9-4. 공유 성장 루프

사용자 A 분석
→ 결과 공유
→ 사용자 B 유입
→ 무료 분석
→ 사용자 B 결과 공유
→ 새로운 사용자 유입

공유 링크의 유입 경로를 추적할 수 있도록 설계한다.

---

## 9-5. 궁합 바이럴

궁합은 단순한 상품이 아니라
두 번째 사용자를 자연스럽게 연결하는 성장 기능으로 활용한다.

사용자 A
→ 상대방 초대
→ 사용자 B 유입
→ 두 사람 관계 분석
→ 필요 시 심층 궁합 구매

개인정보와 상대방 동의 원칙을 준수한다.

---

## 9-6. 콘텐츠 마케팅

명리학 용어 설명만을 중심으로 하지 않는다.

실제 사용자의 고민을 중심으로 콘텐츠를 만든다.

예:

- 올해 이직해도 될까?
- 사업을 시작하기 좋은 시기는?
- 결혼운이 들어오는 시기는?
- 돈이 모이지 않는 이유는?
- 이사하기 좋은 시기는?
- 개업 날짜는 어떻게 정할까?

기본 구조:

사용자의 고민
→ 유용한 정보
→ "내 사주에서는 어떨까?"
→ 운보다 무료 개인 분석

---

## 9-7. 문제별 랜딩 페이지

관심사별 유입 경로를 만든다.

재물 고민
→ 무료 재물 흐름
→ 개인 변화 발견
→ 재물운 심층 분석

직업 고민
→ 무료 직업 변화 확인
→ 변화 시기 발견
→ 직업 / 사업운 분석

관계 고민
→ 무료 관계운 확인
→ 관계 변화 발견
→ 연애 / 궁합 분석

측정:

- 평균 구매 금액
- 반복 구매율
- 고객 생애 가치

광고부터 결제 상품까지
하나의 사용자 고민으로 연결한다.

# PART 11. 개인정보·신뢰·안전 원칙

- 사용자 동의 없이 분석 결과를 광고에 사용하지 않는다.
- 마케팅 알림과 서비스 알림을 구분한다.
- 탈퇴 및 데이터 삭제 구조를 마련한다.
- 구매한 디지털 자산의 보관 정책을 명확하게 한다.
- 관상과 손금 등 이미지 기반 콘텐츠는 과도한 사실 단정을 피한다.
- 건강, 범죄성, 능력 등 중대한 특성을 이미지로 단정하지 않는다.
- 전통 문화 및 해석 체계의 성격을 사용자에게 적절히 안내한다.

# FINAL PRODUCT PRINCIPLE

운보다는

무료 사주로 신뢰를 만들고,

계산 엔진으로 사용자의 현재 운과 변화를 분석하며,

개인 맞춤 추천으로
지금 필요한 분석을 발견하고,

고민별 단품 심층 분석으로
첫 번째 유료 가치를 제공하며,

AI 명리 상담과 운의 변화 추적으로
지속적인 관계를 만들고,

필요한 사용자에게
선택형 구독을 제공하며,

궁합·택일·작명·개명·이사·개업·타로·풍수 등으로 확장하는

AI 기반 통합 운명 분석 플랫폼을 구축한다.

유입

→ 개인화
→ 단품 구매
→ 만족
→ 재방문
→ 반복 구매
→ 선택형 구독
